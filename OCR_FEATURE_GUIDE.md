# OCR文本识别功能使用指南

## 功能概述

本系统已集成了基于Tesseract.js的OCR文本识别功能，支持自动识别以下文件类型：

| 文件类型 | 识别内容 | 自动应用 |
|--------|--------|--------|
| **身份证** | 身份证号、出生日期、年龄、姓名、住址 | ✅ 可一键回填任务页 |
| **病历** | 诊断信息、就诊日期 | ✅ 识别后展示结果 |
| **发票** | 发票金额、非医保用药金额、发票号、日期 | ✅ 自动添加到损失项 |

## 快速开始

### 第一步：打开单证管理页面
1. 从案件详情页（TaskDetail）底部点击 **"上传附件"** 按钮
2. 或直接访问 `/attachments/:caseId` 路由

### 第二步：上传文件
1. 点击顶部 **"批量上传"** 按钮
2. 选择一张或多张图片/PDF文件
3. 文件会自动显示在缩略图网格中

### 第三步：触发OCR识别
1. 点击任意图片缩略图选中文件
2. 在右侧预览区点击 **"识别文本"** 按钮
3. 等待识别完成（首次加载需要下载语言包，约50MB）

### 第四步：查看和应用识别结果
1. 识别完成后，右侧显示结构化的识别结果卡片
2. 验证识别内容是否准确
3. 点击 **"应用到表单"** 按钮

## 各文件类型识别详解

### 📋 身份证识别

**支持位置**：身份证明 (identity) 文件夹

**识别内容**：
```
姓名：张三
身份证号：110101199003019999
出生日期：1990-03-01
年龄：34岁
住址：上海市浦东新区张江路88号
```

**应用说明**：
- 识别结果在预览区展示
- 通过 `window.message` 事件传回表单页面
- 自动回填伤者姓名、身份证号、出生日期、年龄
- 自动将身份证住址填入任务页“户籍地”字段

### 🏥 病历识别

**支持位置**：
- 医疗记录（三者 / 骑手）
- 医疗费用清单（三者 / 骑手）
- 鉴定报告（三者 / 骑手）

**识别内容**：
```
诊断：右腿骨折
日期：2026-04-07
```

**应用说明**：
- 通过正则表达式匹配"诊断""医嘱"等关键词
- 支持多种日期格式（YYYY-MM-DD / YYYY/MM/DD / YYYY年MM月DD日）
- 识别结果用于提示查勘员

### 📄 发票识别

**支持位置**：
- 医疗费发票（三者 / 骑手）
- 维修单据 (repair) 文件夹

**识别内容**：
```
金额：￥5000.00
非医保用药：￥800.00
发票号：No.20260407001
日期：2026-04-07
```

**应用说明**：
- ✨ **特殊功能**：自动将识别的发票金额添加到"损失项目"表格
- 原理：通过 `window.message` 事件将发票信息传给TaskDetail
- 系统自动创建新的医疗费用项目记录

## 技术细节

### OCR处理流程

```
上传图片
  ↓
转换为Base64 Data URL
  ↓
Tesseract.js加载chi_sim语言包
  ↓
提取图片中的全部文本
  ↓
用正则表达式提取关键信息
  ↓
结构化为JSON对象
  ↓
在UI中展示识别结果
```

### 支持的正则模式

| 识别项 | 正则模式 | 示例 |
|-------|--------|------|
| 身份证号 | `\d{18}` | 110101199003019999 |
| 出生日期 | 从身份证号第7-14位 | 1990-03-01 |
| 诊断 | `/(?:诊断\|医嘱)[：:]*\s*([^\n。，,;]+)/` | 右腿骨折 |
| 金额 | `/[￥人民币]*\s*(\d+[.,]\d{2}\|\d+)/` | ￥5000.00 |
| 自费金额 | `/(?:自费\|自付\|非医保)[：:]*\s*(\d+[.,]\d{2}\|\d+)/` | ￥800.00 |

## 常见问题

### Q1: 为什么首次使用很慢？
**A**: Tesseract.js需要下载中文简体(chi_sim)语言包，约50MB。首次下载后会缓存在浏览器本地存储中。

### Q2: 识别准确率如何？
**A**: 准确率取决于：
- 图片质量（清晰度、光线、角度）
- 文字类型（印刷体 >手写体）
- 识别模型（chi_sim支持简体中文）

建议对识别结果进行人工验证后再应用。

### Q3: PDF格式的文件能识别吗？
**A**: 当前实现对PDF显示占位符。后续可集成pdf2image库进行处理。

### Q4: 支持其他语言识别吗？
**A**: 可以，Tesseract支持100+种语言。需要在`initWorker`中加载对应语言包。

### Q5: 识别结果如何持久化？
**A**: 当前结果存储在state中。可通过以下方式持久化：
- 保存到后端数据库
- 存储在localStorage浏览器本地存储
- 添加到附件的metadata字段
